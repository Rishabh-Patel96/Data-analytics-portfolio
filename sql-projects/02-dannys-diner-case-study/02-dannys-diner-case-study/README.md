# Danny's Diner SQL Case Study

## Project Overview

This project analyzes customer purchasing behavior for a restaurant called **Danny's Diner** using SQL.

The objective of the case study is to explore customer ordering patterns, spending behavior, and loyalty program performance.

This project demonstrates practical SQL skills used in data analysis and business intelligence workflows.

---

## Problem Statement

Danny wants to use the data to answer a few simple questions about his customers, especially about their visiting patterns, how much money they’ve spent and also which menu items are their favourite. Having this deeper connection with his customers will help him deliver a better and more personalised experience for his loyal customers.

He plans on using these insights to help him decide whether he should expand the existing customer loyalty program - additionally he needs help to generate some basic datasets so his team can easily inspect the data without needing to use SQL.

## Business Questions

The analysis answers the following questions:

1. What is the total amount each customer spent at the restaurant?
2. How many days has each customer visited the restaurant?
3. What was the first item from the menu purchased by each customer?
4. What is the most purchased item on the menu?
5. Which item was the most popular for each customer?
6. Which item was purchased first after becoming a member?
7. Which item was purchased just before becoming a member?
8. What is the total items and amount spent before becoming a member?
9. How many loyalty points would each customer have?
10. How many points do customers have at the end of January considering the bonus period?

---

## Dataset

The project uses three tables:

- **sales** – records of customer purchases
- **menu** – product information and prices
- **members** – loyalty program membership data

---

## SQL Techniques Used

This project demonstrates:

- Aggregations (`SUM`, `COUNT`)
- `GROUP BY` analysis
- Window functions (`ROW_NUMBER`, `RANK`)
- Conditional logic (`CASE`)
- Date filtering
- Joins between relational tables
